Урок 1. Микроконтроллер Iskra JS и дополнительные электронные компоненты

Кто-нибудь из вас знает, что такое робот и что он умеет делать? Как он может что-либо увидеть?

Идея создания роботов всегда увлекала человечество, ведь их создавали по человеческому подобию. На данный момент роботы тесно сопряжены с нашей жизнью. Они облегчают нашу повседневную жизнь, помогая по домашней работе, также они спасают нашу жизнь (роботы-сапёры).

Робототехника – относительно новое и быстро развивающееся научное направление, необходимое для освоения современных сфер и областей деятельности человека. Помимо этого робототехника помогает автоматизировать современное производство, которое направлено на повышение эффективности работы.

Отцом робототехники был Джозеф Энгельбергер, именно он и Джордж Девол в 1956 году создали первый прототип коммерческого робота и основали первую в мире фирму, которая производила промышленных роботов.

Следует усвоить законы робототехники:

1. робот никогда не должен причинить вред человеку и не может допустить, чтобы был причинен вред человеку кем-либо другим;
2. все команды, которые дает человек, выполняются беспрекословно, если они не противоречат первому закону;
3. самостоятельная безопасность робота не должна противоречить двум предыдущим законам.

После того, как вы узнали законы робототехники, можно ознакомиться с микроконтроллером Iskra JS, принципами его работы, компонентами.

Iskra JS – это своеобразный мозг робота, маленький компьютер, в котором выполняются различные алгоритмы.

Плата Iskra JS имеет возможность подключения различных дополнительных модулей, которые расширят просторы для создания интересных новых устройств. Плата способна измерять напряжение и выдавать его. Данной функции достаточно для того, чтобы взаимодействовать с окружающим миром: считка различных сенсоров, выдача команд на светодиоды, дисплеи моторы и так далее.

Для описания «поведения» устройства необходимо запрограммировать этот маленький компьютер. Данная плата думает на языке программирования JavaScript, но стоит отметить, что есть особенности программирования, к примеру, отсутствие регулярных выражений (поиск).

Помимо микроконтроллера Iskra JS понадобятся следующие дополнительные электронные компоненты:

1.Troyka Shield – специальная плата расширения, помогающая подключить большое количество периферийный устройств (датчики, сенсоры) через специальные провода для соединения сенсора и модуля.

Данная плата нужна для объединения всех модулей серии и является центральным звеном. По периметру платы можно увидеть тройки контактов «питание (S) – земля (V) – сигнал (G)». Они подключены с линиями управляющей платы, в нашем случае с платой Iskra JS. Соединение происходит следующим образом: сигнал с аналоговым или цифровым пином, питание с рабочим напряжением, земля с землёй.

2. Шлейфы.Представлен стандартный трехпроводной шлейф (рис. 14), который необходим для соединения модуля и платы. По сути, это специальный провод, требуемый для соединения.

Для удобного использования обычно подключение проводов используют для следующих назначений: черный – земля, желтый – сигнал, красный – питание. Данный шлейф используется для работы с напряжением до 50 В и током до 1А.

3. Светодиод. Особенностью данного модуля является большое потребление тока, микроконтроллеры не могут дать такое количество тока напрямую. На борту модуля имеется транзистор, через который происходит управление светодиодом. За счет этого модуль будет совместим с сигналами не только на уровне 5В, но и на 3.3 В. Стоит осторожно обращаться с этим светодиодом, так как светодиодный модуль очень яркий и на него не стоит смотреть с близкого расстояния, это может быть очень вредным для глаз.

4. Кнопка.Кнопка является цифровым датчиком: пока кнопка не зажата, датчик показывает логическую 1, когда кнопка зажата, датчик показывает логический 0.

5. Потенциометр.Потенциометр используется для ручного задания параметров, таких как время таймера, яркость или контрастность дисплея, скорость вращения двигателя, высота звука.

Модуль с потенциометром является аналоговым сенсором, сообщающим микроконтроллеру положение ручки потенциометра. Если нужно измерить напряжение, потенциометр также подойдет. Выходной результат работы – аналоговый сигнал с уровнем напряжения, который прямо пропорционален углу поворота ручки потенциометра.

6. Зуммер. Слово «зуммер» («summen») переводится с немецкого – жужжать. По сути это – звукоизлучающее устройство, которое используется для какого-либо сигнала.

Для создания звука пьезоизлучатель переводит переменное напряжение в колебание. Зуммер может создавать тревожные звуки при сигнализации, или играть какую-либо мелодию.

7. Датчик освещенности основан на фоторезисторе GL5528. Фоторезистор – компонент, который меняет сопротивление в зависимости от количества света.

Фоторезистор GL5528 представляет собой полупроводниковый радиоэлемент, меняющий свое сопротивление в зависимости от освещения. Он нужен для регистрации видимого света с высоким порогом чувствительности. При помощи датчика освещенности устройство может отличить день от ночи, тень от света, солнечную от пасмурной погоды.

Урок 2. Устройство Iskra JS.Начало работы с JavaScript

На прошлом уроке мы с вами познакомились с микроконтроллером Iskra JS и различными электронными компонентами (светодиод, кнопка, потенциометр, зуммер, датчик освещенности и другие), изучили их более подробно. Теперь разберем более подробно само устройство платы Iskra JS. Это очень важно, так как практически каждый элемент, который находится на плате, несет свою функцию.

1 – разъём для питания платы от розетки или аккумулятора.

2 – светодиоды:

ON – загорается при поступлении питания на плату;
LED1 – используется по собственному усмотрению;
BUSY – загорается при произведении вычислении микроконтроллером, а также передачи данных.

3 – Разъём micro-USB – при помощи данного разъема происходит загрузка программы, передача данных на компьютер, а также питание платы.

4 – Кнопка RESET нужна для перезагрузки платы, программа, которая была сохранена последней, будет исполняться с самого начала.

5-9, 11-13 – Колодки с контактами вдоль длинных сторон — это входы и выходы платы. Их называют пинами:

6, 7, 8, 11, 12 – Пины P0…P13 и A0…A5 необходимы при взаимодействии с модулями, платами расширения и другими периферийными устройствами. Их называют портами GPIO (General Purpose Input and Output) или просто портами. Нужны для определения напряжения на них в режиме входа;

7, 11, 12 – Пины A0…A5 и отмеченные А измеряют точную величину входного напряжения. К ним можно подключить аналоговые сенсоры;

6, 7, 8, 12 – Отмеченные тильдой (~) переключают выходное напряжение между 0 и 3,3 вольтами 1000 раз в секунду. Данная функция нужна для контроля яркости светодиодов, мощности магнитов и так далее;

5, 6, 9, 11 – Пины DAC, RX, TX, SPI, SDA, SCL необходимы при определенной работе с некоторыми модулями.

10 – Кнопка BTN1 используется по собственному усмотрению.

13 – Пины питания:

3.3V и IORef выдают ровные и родные для Iskra JS 3,3 вольта;
5V выдаёт 5 вольт для модулей, которым не хватает обычных 3,3 вольта;
VIN выдаёт входное напряжение;
GND – земля платы, начало 0 В.

14 – Джампер с двумя положениями: VIN3+5V (обычный режим), VIN3+VIN (питает низковольтный аккумулятор).

Язык программирования JavaScript был создан в 1995 году Бренданом Эйхом. Для начала разберем основы программирования на JavaScript. JavaScript – популярный язык программирования. Он простой, выразительный и гибкий. JavaScript обычно используют для создания веб-приложений, но его также может исполнять и Iskra JS.

Давайте посмотрим, как отреагирует плата на обычные слово. В окне консоли введем "hello!" (очень важно не забыть кавычки) и нажмем Enter, чтобы отправить строку на плату. Мгновение спустя можно увидеть ответ от платы

>'hello!'

>="hello!"

Это означает, что всё настроено правильно и плата готова к чему-то более сложному.

Теперь узнаем с вами об арифметических операциях (операторах). Они принимают численные значения и возвращают число.

Стандартные операторы:

сложение (+) – возвращает сумму чисел или может объединить строки;
вычитание (-) – оператор вычитает один операнд из другого и возвращает их разницу;
умножение (*) – производит произведение операндов;
деление (/) – происходит деление операндов.

Попробуем дать различные инструкции плате и увидим результат их выполнения

>20-12+13

=29

>6+7-3*6/5

=9.4

Во втором случае получается нецелое число, такие числа поддерживаются, поэтому будет выведен результат.

Стоит отметить, что есть приоритет в операциях: сначала выполняется умножение или деление, затем сложение или вычитание. Чтобы скорректировать приоритет используются круглые скобки

>6+(7-3)*6/5

=10.8

Переменные JavaScript используют для хранения значений или выражений. Они могут иметь короткое имя (Х, Y), или же более длинное (led, button). Есть важные правила при создании имен переменных:

чувствительность к регистру (х и Х это разные переменные);
переменная не может быть зарезервированным словом;
имена могут начинаться с буквы или подчеркивания.

Бывают различные типы переменных:

числовые над ними можно выполнять различные операции: сложение умножение и так далее;
строковые – какая-либо строка. Стоит отметить, что строковый тип переменных указывается в кавычках, нет особой разницы, какие именно кавычки использовать одинарные или двойные, интерпретатор поймет;
булевые (логические). – данные переменные равны либо истине (true), либо лжи (false). Обычно такие переменные используются в условных операторах, с которыми вы познакомитесь позднее.

Создание переменных в языке программирования JavaScript обычно называют «объявлением» переменной. Для этого используется ключевое слово var (variable). Теперь перейдем в другую часть окна и напишем небольшую программу.

var y; //пустая переменная

var x=5; // переменной присвоено значение 5 – числовая переменная

var MyName = "Julia"; // переменная строкового типа

var button = false; // булевая переменная

print (y, x, MyName, button); //вывод на экран

Вывод:

undefined 5 Julia false

Также можно решать простые выражения

var x=4;

var y =10;

print (x + y * y + 5);

Вывод:

109

x = y + 3;

print (x);

Вывод:

13

print (x + y * y + 5);

Вывод:

118

Теперь узнаем с вами о функциях. Функция обычно представляет собой группу выражений или выражение, которые нужны для выполнения определенной задачи. К примеру, есть различные системные функции, которые могут:

вернуть заводской номер платы – getSerial();
вернуть время, прошедшее с включения платы – getTime();
сбросить интерпретатор, то есть очищает память – reset();
и многие другие.

Каждый раз, когда будет вызываться данная функция, будет меняться время. Время показывается в виде вещественного числа в секундах. Обратим внимание на плату, загорелся светодиод LED1. Разберем, что именно мы сделали, чтобы загорелся светодиод.

>LED1.write(1)

=undefined

Нужно обратиться к объекту LED1 и затем вызвать метод write. Метод – действия, выполняющиеся над объектами. В скобках у нас написана 1 – это параметр, который передается методу.

>LED1.write(0)

=undefined

Теперь светодиод погас. У вас возник вопрос, что такое undefined. Это специальное значение, которое возвращает отсутствие значения у переменной, или просто отсутствие чего-либо. К примеру, мы с вами разбирали функции, каждая из них что-то возвращает – getSerial() вернет заводской номер платы, а вот LED1.write не может вернуть ничего, поэтому выводится undefined.

Урок 8. Операторы сравнения и логические операторы. Условный оператор if

Для начала мы с вами узнаем, какие бывают логические операторы и операторы сравнения.

Операторы и их значения
ОператорЗначение
a < ba меньше b
a <= ba меньше или равно b
a > ba больше b
a >= ba больше или равно b
a = = = ba равно b
a !== ba не равно b
a < b && a >= ca меньше b и a больше либо равно c
a < b || a >= ca меньше b или a больше либо равно c
a a истинно или не ноль
a || b< ba истинно или не ноль или b истинно или не ноль
!a && !b < ba ложно или ноль и b ложно или ноль

Пояснения:

1. Оператор ! – отрицание, истина превращается в ложь, ложь в истину.
2. Оператор && – логическое «И» (конъюнкция), когда оба операнды истинны, возвращается истинное значение.
3. Оператор || – логическое «ИЛИ» (дизъюнкция), когда хотя бы один из операндов истинный, возвращается истинное значение.

У логических операторов следующий приоритет: отрицание, конъюнкция, дизъюнкция. Также как и арифметических операторов скорректировать приоритет можно с помощью круглых скобок.

Также как и в случае с арифметическими операторами для коррекции приоритета используются круглые скобки. Если одна пара скобок вложена в другую пару скобок, то сначала вычисляется значение во внутренних скобках.

Данные операторы нам сегодня понадобятся на уроке. Итак, теперь мы с вами познакомимся с другим оператором – условный оператор или по-другому условное ветвление. Данный оператор нужен для выполнения различных команд в зависимости от заданного условия. Оператор if(...) сначала вычисляет условие, которое задано в скобках, а затем есть один или два варианта исхода событий.

Существует два вида ветвления: полное и неполное ветвление .

При полном ветвлении, если условие верно (true), то выполняется блок кода, который идет в конструкции if, если условие неверно (false), то выполняется условие конструкции else.

При неполном ветвлении нет инструкции else, поэтому блок кода инструкции if может выполниться только, если условие истинно.

Именно для записи различных условий используются логические операторы и операторы сравнения.

Пример 1.

var x = 10;

var y = 5;

if (y < x ) print (' y < ', x);

Вывод:

Пример 2.

var x = 10;

var y = 5;

if (y > x ) print (' y > ', x);

Вывод:

не будет вывода, так как y меньше

Пример 3.

var x = 5;

var y = 10;

if (y < x ) print (' y < ', x);

else print (' x < ', y);

Вывод:

Если нужно выполнить несколько команд в инструкции, то стоит взять команды в фигурные скобки {}. Они объединяют выполнение команд.

Пример 4.

var x = 5;

var y = 10;

if (y < x ) {

print (' x > y ');

print ( x, y);

}

else{

print (' x < y ');

print ( x, y);

}

Урок 11.Цикл с предусловием while

Цикл – это управляющая инструкция, позволяющая повторять выполнение программного кода определённое количество раз. Каждое отдельное исполнение инструкций в теле цикла называется итерацией.

Синтаксис:

while (условие) {

//тело цикла

}

В данном синтаксисе нет ни переменной для итерации, ни каких-либо действий после итерации как в цикле for. Цикл while в JavaScript выполняется, пока истинно определенное условие. После того, как условие расценивается как ложное, тогда цикл while прекращается.

Пример 1.

var i=9;

while (i>0){

print (i);

i=i-1; или i--

}

Вывод:

Пример 2.

var i=9;

while (i<15){

print (i);

i=i+1; или i++

}

Вывод:

Пример 3.

var i=9;

var button = false;

while (i<15 && button === false){

print (i);

i=i+1; или i++

}

Вывод:

Пример 4.

var i=9;

var button = true;

while (i<15 && button === false){

print (i);

i=i+1; или i++

}

Цикл не выполнится

Урок 10. Итерационный цикл for

Оператор цикла for называют цикл с параметром. Цикл с параметром используется в том случае, когда требуется выполнить заданное количество шагов цикла.

Синтаксис:

var i;

for(i=0; условие; i++) { //переменную можно задать сразу же в самом цикле

блок кода;

}

В качестве условия, в циклах JavaScript, обычно выступают операции сравнения.

Пример

for(var i=0; i<5; i++) {

print("Привет!");

}

Данный пример выведет 5 раз слово привет в столбик

Давайте разберем более подбробно, что произошло. Когда интерпретатор находит в коде цикл for, сначала выясняет значение переменной i (i=0), далее проверяет условие (i<5), если условие подходит вернёт true и запустит код, который размещён в фигурных скобках { }. После чего выполнится арифметическая операция (i++). Вот так интерпретатор выполнил один раз цикл (один повтор = итерация).

В данном случае произойдет 5 итераций и как только i станет равной 5, то условие (i<5) вернёт false и выполнение цикла завершится.

Урок 12. Цикл с предусловием while

Всем давно известно как можно найти НОК (наименьшее общее кратное) и НОД (наименьший общий делитель) двух чисел. Давайте вспомним алгоритм нахождения.

Наибольшим общим делителем (НОД) двух данных чисел a и b является наибольшее число, на которое оба числа a и b делятся без остатка.

Наименьшим общим кратным (НОК) двух натуральных чисел называется наименьшее натуральное число, которое само делится нацело на каждое из этих чисел.

Для нахождения НОД будем использовать алгоритм Евклида:

Программный код:

var random = require ('@amperka/hw-random');

var c= random.int (1,50);

var f= random.int (51,100);

var a,b;

function NOD (a,b){

while (a != b){

if (a > b){

a=a- b;

} else {

b = b-a;

}

}

return a;

}

var x = NOD (c,f);

print (c,'',f);

print (x);

Урок 13.Строковые переменные

Сегодня мы с вами изучим более подробно такой тип данных, как строковый.

String - класс, который предназначен для работы со строковыми перемнными. Чтобы превратить значение переменной в строку, её нужно выделить кавычками (один из способов задания строковой переменной).

К способам создания переменных строкового типа относят следующие:

1. Присваивание текста

var text = "Hello";

2. Использование оператора new

var text = new String("Hello");

3. Составление из кодов символов

var text = String.fromCharCode(code1 [, … , codeN]);

В классе String имеется множество методов, давайте попробуем некоторые из них. В качестве текста используйте свое имя.

charAt(pos) - вернет символ, который находится на позиции pos.

charCodeAt(pos) - вернет код символа, который находится на позиции pos.

length - вернет длину строки (считая пробелы и невидимые знаки).

slice(start [, end]) - вернет обрезанную строку с начального значения start до конечного значения end (символ на позиции end не включается).

toLowerCase() - вернет строку, преобразованную в нижний регистр

toUpperCase() - вернет строку, преобразованную в верхний регистр

Давайте теперь перейдем к более интересной части урока. Соберем с вами устройство, которое будет генерировать вам пароли по зажатию кнопки
